# おもろい三麻 ミクボイス制作マニュアル

三麻の宣言ボイス4種(リーチ/ロン/ツモ/ペイ)を初音ミク V4X で作るための完全手順書です。
所要時間の目安: 初回30〜40分(2回目以降は15分程度)。

---

## 0. 使うもの

| ツール | 用途 |
|---|---|
| **Cubase LE 13**(デスクトップのショートカット or Steinberg Hub から起動) | DAW本体。Piapro を中で動かし、最後の WAV 書き出しもここ |
| **Piapro Studio**(Cubase の中で開く VSTi) | ミクに歌わせるエディタ。ここに vsqx を読み込む |
| **UR22C**(オーディオインターフェース) | 音のモニター用。**背面USB-C → PC に直接接続**(白LED点灯=OK、点滅=未接続) |
| この フォルダの `*.vsqx` ×4 | 音程・歌詞・発音入りの元データ(叩き台) |

### ファイル対応表

| 元データ | フレーズ | 音のイメージ(ハッピー設計: 全部尻上がり) | 納品ファイル名(重要) |
|---|---|---|---|
| `riichi.vsqx` | リーチ! | り(C5)→ち(E5)へ跳ね上げて伸ばす | `riichi.wav` |
| `ron.vsqx` | ロン! | ろ(C5)→ん(E5)で明るく響かせる | `ron.wav` |
| `tsumo.vsqx` | ツモ! | つ(B4)短く→も(E5)へ大きく跳ねて一番元気に | `tsumo.wav` |
| `kita.vsqx` | ペイ!(北) | ぺ(D5)ぽんと→い(F#5)キラッと(かわいい担当) | `kita.wav` |

- BPM128・全フレーズ「終わりの音が高い」上行形 = 明るく楽しい掛け声になる設計です

## 1. プロジェクト準備(初回のみ・約5分)

1. **Cubase LE 13 を起動** → Steinberg Hub が開く
2. 「**空白**」(Empty)を選び、プロジェクトフォルダは任意の場所(例: `ドキュメント\Cubase\sanma_voice`)→「**作成**」
3. 音が出るか確認:
   - 上部メニュー「**スタジオ → スタジオ設定 → オーディオシステム**」
   - ASIOドライバーが「**Yamaha Steinberg USB ASIO**」になっているか確認(なっていなければ選び直す)
   - ※ Generic Low Latency ASIO だとヘッドセットがモノラルになる既知問題あり
4. **Piapro Studio トラックを追加**:
   - トラック一覧の何もない所を**右クリック → 「トラックを追加」→「インストゥルメント」**
   - 「インストゥルメント」欄のプルダウンから **Piapro Studio VSTi** を選択 →「トラックを追加」
   - ※ ここで E02 エラーが出た場合は過去に直した実績あり(Piapro再インストール)。Claude に「E02出た」と言ってくれれば手順を出します
5. 追加したトラックの左ゾーンにある**鍵盤マーク(インストゥルメントを編集)**をクリック → **Piapro Studio の画面が開く**

## 2. 元データの読み込み(フレーズごと・約2分)

1. **Piapro Studio の画面内**の上部メニュー「**ファイル → インポート**」
2. この`voice_source`フォルダから **`riichi.vsqx`** を選択
3. シンガー(歌声)を聞かれたら / または歌わないときは:
   - Piapro 内のトラック左端の**シンガーアイコンをクリック → 「シンガーの変更」**
   - **MIKU V4X の好きなボイスバンク**を選択(掛け声には **Original** か **Solid** がハッキリして向いてます。かわいく寄せるなら **Sweet**)
4. ピアノロールに 2つのノート(例: 「り」「ち」)が **2小節目の頭**から置かれていれば読み込み成功
5. **スペースキーで再生**して聞いてみる(再生位置が遠くにいたら テンキーの `.` で先頭に戻る)

> 🔧 **vsqx が読み込めなかったら**(このファイルは手製なので万一の保険):
> - 同名の `.mid` を同じ手順でインポート → ノートを全選択(Ctrl+A)→ 右クリック「**歌詞の流し込み**」→ 「りち」(riichi) / 「ろん」 / 「つも」 / 「ぺい」 と入力
> - それもダメなら**手打ちが最速**: 鉛筆ツールで2小節目頭にノートを2つ置き(音の高さは下の表)、ノートをダブルクリックして歌詞を1文字ずつ入力
>
> | フレーズ | 1音目 | 2音目 |
> |---|---|---|
> | リーチ | り = C5・短く | ち = E5・伸ばす |
> | ロン | ろ = C5 | ん = E5・長め |
> | ツモ | つ = B4・ごく短く | も = E5・長め(大きく跳ねる) |
> | ペイ | ぺ = D5・短く | い = F#5・キラッと |

## 3. ミクらしく調整(お好みで)

元データはあくまで叩き台です。よく効くパラメータ:

- **音程・ノートの長さ**: ピアノロールでドラッグ。掛け声なので多少大げさな高低差の方が映えます
- **VEL(ベロシティ)**: 上げると子音が短くキレ良く(「チ!」の切れ味)
- **DYN(ダイナミクス)**: 音量の抑揚。フレーズ末尾を少し下げると自然
- ビブラートはオフにしてあります(掛け声には不要)
- **4ファイルで声・パラメータの条件を揃える**こと(1つだけ声が違うと ゲーム内で浮きます)

## 4. WAV 書き出し(フレーズごと・約2分)

> ⚠️ Piapro Studio 単体では書き出せません(メニューがグレーアウト)。**必ず Cubase 側から**行います。

1. **書き出し範囲(ロケーター)を設定**:
   - Cubase のルーラー(小節番号の帯)上を**ドラッグ**して、**1小節目の4拍目あたり 〜 3小節目の頭**を範囲指定
   - (子音が2小節目のわずかに手前から鳴るため、2小節目ちょうどからだと頭が切れることがあります)
2. 上部メニュー「**ファイル → 書き出し → オーディオミックスダウン**」
3. 設定:
   - **ファイル名**: `riichi`(対応表の名前。あとで拡張子 .wav が付きます)
   - **保存先(パス)**: 分かりやすい場所(例: デスクトップの `sanma_voice` フォルダ)
   - **ファイル形式**: Wave ファイル
   - **サンプリングレート**: 44.100 kHz / **ビット深度**: 16 bit
   - ノーマライズ等のチェックは**オフのまま**
4. 「**オーディオミックスダウン実行**」(または「書き出し」)ボタン

## 5. 残り3フレーズ(2通りのやり方)

**やり方A: 1プロジェクトに全部読み込む(おすすめ)**
1. Piapro Studio で「ファイル → インポート」を繰り返し、`ron.vsqx` `tsumo.vsqx` `kita.vsqx` も読み込む(Piapro内にトラックが増えていく)
2. 書き出したいフレーズのトラックだけ **S(ソロ)** ボタンを点灯(他は鳴らさない)
3. 手順4の書き出しをファイル名を変えて繰り返す(ロケーター範囲は同じでOK)

**やり方B: 素直に1フレーズずつ** — riichi を書き出したら Piapro のトラックを削除して次を読み込む。混乱しにくいですが保存はこまめに。

## 6. 納品

1. 4ファイルの名前を確認: `riichi.wav` `ron.wav` `tsumo.wav` `kita.wav`(全部小文字)
2. **Box の `claude/omoroi-sanma/voice/`**(このフォルダの隣、作成済み)にコピー
3. Claude に「**ミクボイス置いたよ**」と一言 → repo 組み込み・「VOICE: 初音ミク」クレジット追加・動作確認・リリースまで実施します

## トラブルシュート(過去の実績から)

| 症状 | 対処 |
|---|---|
| 音が出ない | UR22C の USB を確認(背面USB-C→PC直挿し、白LED**点灯**)。スタジオ設定で Yamaha Steinberg USB ASIO を選択 |
| Piapro が E02 で開かない | 再インストールで直した実績あり。Claude に声かけで手順提示 |
| vsqx インポート失敗 | 手順2の🔧参照(mid → 歌詞流し込み、または手打ち) |
| 書き出した WAV の頭が切れる | ロケーター左端を1小節目4拍目より前に広げる |
| 4ファイルの音量がバラバラ | 同一プロジェクト・同一設定で書き出す(やり方A)。ミクのDYNも揃える |
